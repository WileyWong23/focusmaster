## 一、产品概述

### 1.1 产品定位
**专注大师**是一款以游戏化段位升级为核心激励机制的专注计时应用，帮助用户设定专注目标、跟踪累计时长、通过从青铜到王者的阶梯式段位体系激励持续专注。

### 1.2 核心价值
- **目标驱动**：用户自定义专注目标，赋予每次专注以意义
- **段位激励**：7级段位体系（青铜→王者），阶梯式升级制造成就感
- **沉浸专注**：深空主题 + 极简界面，减少干扰，专注当下

### 1.3 技术方案
- 前后端分离，H5 单页应用
- 后端：Spring Boot + H2 嵌入式数据库
- 前端：原生 HTML/CSS/JS（嵌入 Spring Boot 静态资源）
- 后续可通过 Capacitor 等工具将 H5 封装为 Android APK（不额外增加安卓开发角色）

### 1.4 设计风格
- 主题：**Celestial Flow（深空星流）**
- 深色沉浸式界面，玻璃拟态(Glassmorphism)效果
- 色彩层级替代边框，通过明暗深度区分信息层级
- 字体：Manrope（标题）+ Inter（正文/标签）

## 三、功能需求详述

### 3.1 首页 (Home)

**页面用途**：核心操作入口。展示当前进行中目标的状态，设置本次专注时长，开始专注计时。

**适用场景**：
- 有进行中目标时：展示目标信息、段位、设置时长并开始专注
- 无进行中目标时：引导用户去目标页创建

**页面布局（有目标时，从上到下）**：

```
┌─────────────────────────────┐
│       顶部区域               │
│  目标名称 + 段位标签         │  如"考研" + [黄金] 标签
│  已累计 XXh / 目标 XXh      │  进度文字
├─────────────────────────────┤
│                             │
│        开始专注              │  ★ 大号渐变圆形按钮（突出）
│       (大按钮)               │  脉冲动画，视觉焦点
│                             │
├─────────────────────────────┤
│       时长选择               │
│  ◄  21  22  [23]  24  25  ► │  横向滑动选择器（紧凑一行）
│                    分钟      │  范围: 5~180分钟，间隔1分钟
├─────────────────────────────┤
│  ┌─段位条──────────────────┐ │
│  │青铜 白银 黄金 白金 钻石..│ │  横向7段位进度指示器
│  └─────────────────────────┘ │  当前段位高亮
├─────────────────────────────┤
│     首页  目标  我的         │  底部导航栏
└─────────────────────────────┘
```

**页面元素详细**：

| 区域 | 元素 | 说明 |
|------|------|------|
| 顶部 | 目标名称 | 大号标题字体，如"考研" |
| 顶部 | 段位标签 | 胶囊标签，显示当前段位名称，使用段位对应色彩 |
| 顶部 | 累计进度 | "已累计 XXh / 目标 XXh"，小号灰色文字 |
| 操作区 | **"开始专注"大按钮** | 页面核心焦点，渐变色圆形按钮，带脉冲动画；按钮下方显示当前选择的时长如"25分钟" |
| 时长选择 | **时间滚轮 (Time Picker Wheel)** | 滚轮式选择器，直接嵌入页面 |
| 段位条 | 横向7段位进度指示器 | 完整展示7个段位，已达到的高亮，当前段位特别标记 |
| 底部 | 底部导航栏 | 首页(高亮)、目标、我的 |

**时间滚轮设计规范**：

| 属性 | 说明 |
|------|------|
| 类型 | 横向滑动选择器（类似横向滚轮/刻度尺） |
| 最小值 | 5 分钟 |
| 最大值 | 180 分钟 |
| 步进 | 1 分钟 |
| 默认值 | 25 分钟 |
| 显示 | 选中项居中放大高亮，左右项缩小并透明度降低；紧凑一行展示 |
| 样式 | 深色背景，选中项使用主色调(#81ecff)，未选中项灰色；下方小字"分钟" |
| 交互 | 左右滑动或点击选择 |

**无目标状态**：

| 区域 | 元素 | 说明 |
|------|------|------|
| 中央 | 提示文案 | "还没有进行中的目标" |
| 中央 | "创建目标" 按钮 | 跳转到目标页 |

**交互说明**：
- 滚动时间滚轮选择本次专注时长
- 按钮下方实时显示当前选择的时长
- 点击"开始专注"按钮 → 进入专注计时页（时长已在滚轮确定，无需二次确认）
- 无进行中目标时，"开始专注"按钮禁用，滚轮隐藏

### 3.2 目标页 (Goals)

**页面用途**：管理所有专注目标，查看进行中目标和历史目标。

**页面元素**：

| 区域 | 元素 | 说明 |
|------|------|------|
| 顶部 | "我的目标" 标题 + "+" 新建按钮 | |
| 进行中区域 | "进行中" 标签 | 有进行中目标时显示 |
| 进行中卡片 | 目标名称 + 当前段位 + 段位图标 | 重点展示，带发光边框效果 |
| 进行中卡片 | 进度条 + "XXh / XXh" | 目标进度 |
| 进行中卡片 | "结束目标" 按钮 | 渐变色按钮 |
| 已结束区域 | "已结束" 标签 | |
| 已结束卡片列表 | 目标名称 + 最终段位 + 进度条 | 网格布局(2列) |
| 已结束卡片 | 状态标签(已完成/已放弃) + "重新启动"按钮 | |
| 底部 | 底部导航栏 | 首页、目标(高亮)、我的 |

**新建目标页（子页面）**：

| 区域 | 元素 | 说明 |
|------|------|------|
| 顶部 | 返回按钮 + "新建目标" 标题 | |
| 目标命名 | "目标名称" 标签 + 文本输入框 | placeholder: "例如：考研"，最长20字 |
| 时长选择 | "目标总时长" 标签 | |
| 时长选择 | 预设选项 | 50小时、100小时、200小时 |
| 时长选择 | 自定义输入 | 数字输入 + "小时"单位 |
| 段位预览 | "段位阶梯" 标签 | 动态展示7个段位对应的小时数 |
| 底部 | "创建目标" 渐变按钮 | |

**交互说明**：
- 点击"+" → 进入新建目标页
- 点击"结束目标" → 弹出确认"确定结束「XX」吗？已累计XX小时" → 确认后目标变为已结束
- 点击"重新启动" → 若已有其他进行中目标则提示"请先结束当前进行中的目标"，否则该目标重新变为进行中
- 新建目标时，若有进行中目标则禁止创建，提示"请先结束当前进行中的目标"

**业务规则**：
- 同一时间只能有 **1个** 进行中目标
- "已完成"：累计时长 ≥ 目标总时长
- "已放弃"：用户手动结束且累计时长 < 目标总时长
- 重新启动后已累计时长保留并继续累积

### 4.1 段位定义

| 等级 | 名称 | 色彩 | 图标 |

| 1 | 青铜 | #665500 | stars |

| 7 | 王者 | #ffd709 → #ff6e84 | workspace_premium |

### 4.2 升级阈值（占总时长百分比，非线性递增）

| 段位 | 占比 | 50h | 100h | 200h |
|------|------|-----|------|------|
| 青铜 | 0%（起始） | 0h | 0h | 0h |
| 白银 | ≥5% | 3h | 5h | 10h |
| 黄金 | ≥15% | 8h | 15h | 30h |
| 白金 | ≥30% | 15h | 30h | 60h |
| 钻石 | ≥50% | 25h | 50h | 100h |
| 星耀 | ≥75% | 38h | 75h | 150h |
| 王者 | 100% | 50h | 100h | 200h |

**计算规则**：`所需小时 = max(1, ceil(目标总时长 × 百分比))`，分钟级别精确判定。

## 五、专注计时规则

### 5.1 流程

```
首页 → 滚轮选择时长(5~180min) → "开始专注" → 计时页(倒计时)
                                              ↓
                                      暂停/继续/结束
                                              ↓
                               判断实际时长是否≥5分钟
                                   ↙            ↘
                                 是              否
                                  ↓              ↓
                            计入累计时长      不计入，标记无效
                                  ↓              ↓
                              检查段位变化        ↓
                            ↙          ↘        ↓
                          是            否       ↓
                           ↓            ↓       ↓
                       升级庆祝页   完成页 ← ← ←
                           ↓            ↓
                           → 返回首页 ←
```

### 5.2 累计规则
- 实际专注 ≥ 5分钟：累计时长 += 实际分钟数
- 实际专注 < 5分钟：不计入

### 5.3 段位判定
每次有效专注完成后，根据 `(累计分钟数 / 目标总分钟数)` 的百分比匹配段位阈值。

## 八、UI设计规范摘要

### 色彩
- **主色**: #81ecff（冰蓝）
- **次色**: #ffd709（金黄）
- **三色**: #a68cff（星紫）
- **背景**: #060e20（深空）
- **错误**: #ff6e84（珊瑚红）

### 字体
- **标题**: Manrope（600-800）
- **正文**: Inter（400-600）

### 规则
- 禁止1px实线边框，用背景色层级区分
- 主操作按钮渐变（#81ecff → #00d4ec，135度）
- 卡片玻璃拟态（60%透明 + 12-20px模糊）
